# Naming Skill

## When to use

Use this skill when:
- Creating new classes, services, controllers, or other PHP files
- Naming database columns, tables, or migrations
- Creating new skills, rules, commands, or guidelines
- Naming branches, commits, or PRs
- Naming routes, API endpoints, or URL segments
- Unsure about the correct naming pattern for anything in the project

## PHP Class Naming

### Controllers (Single Action)

```
{Action}{Entity}Controller
```

| Action | Example | HTTP |
|---|---|---|
| Create | `CreateProjectController` | POST |
| List | `ListProjectsController` (plural!) | GET |
| Show | `ShowProjectController` | GET |
| Update | `UpdateProjectController` | PATCH |
| Delete | `DeleteProjectController` | DELETE |
| Restore | `RestoreProjectController` | GET |

### FormRequests

```
{Action}{Entity}Request
```

Examples: `CreateProjectRequest`, `ListProjectsRequest`, `UpdateProjectRequest`

### API Resources

| Pattern | When | Example |
|---|---|---|
| `{Entity}Resource` | Full entity | `ProjectResource` |
| `{Entity}MinimalResource` | Lightweight for lists | `ProjectMinimalResource` |
| `Simple{Entity}Resource` | Embedded in other resources | `SimpleUserResource` |
| `{Entity}StatisticsResource` | Aggregated data | `ProjectStatisticsResource` |

### Services

```
{Domain}Service
```

Examples: `ImportService`, `NotificationService`, `CustomerSyncService`

- Singular noun + `Service` suffix
- Describes the domain, not the action
- No `I` prefix for interfaces: `CustomerSync`, not `ICustomerSync`

### DTOs

```
{Entity}{Purpose}Dto
```

Examples: `ProjectCreateDto`, `ImportRowDto`, `CustomerSyncResultDto`

### Jobs

```
{Action}{Entity}Job
```

Examples: `SyncCustomerJob`, `ProcessImportJob`, `SendNotificationJob`

### Events

```
{Entity}{PastTense}
```

Examples: `ProjectCreated`, `ImportCompleted`, `CustomerSynced`

### Enums

```
{Entity}{Concept}
```

Examples: `ImportStatus`, `ProjectType`, `Queue` (for queue names)

- PascalCase class name
- UPPERCASE cases: `case PENDING;`, `case IN_PROGRESS;`
- Use backed enums for DB values: `case PENDING = 'pending';`

### Policies

```
{Entity}Policy
```

Examples: `ProjectPolicy`, `ImportPolicy`

### Interfaces / Contracts

```
{Capability}
```

- Namespace: `App\Contracts\{Subdomain}\`
- No `Interface` suffix: `SmsProvider`, not `SmsProviderInterface`

## Database Naming

| Element | Convention | Example |
|---|---|---|
| Tables | `snake_case`, plural | `projects`, `import_rows` |
| Columns | `snake_case` | `created_at`, `user_id`, `is_active` |
| Foreign keys | `{entity}_id` | `project_id`, `customer_id` |
| Booleans | `is_` or `has_` prefix | `is_active`, `has_permission` |
| Timestamps | `_at` suffix | `synced_at`, `imported_at` |
| Pivot tables | Alphabetical, singular | `project_user`, `permission_role` |
| Indexes | `{table}_{columns}_index` | Auto-generated by Laravel |

## Route Naming

| Element | Convention | Example |
|---|---|---|
| URL segments | kebab-case, plural | `/api/v1/import-uploads` |
| Route names | dot notation | `v1.import-uploads.index` |
| Route parameters | camelCase | `{importUpload}` |
| API versioning | Prefix | `/api/v1/`, `/api/v2/` |

## Git Naming

### Branches

```
{type}/{ticket-id}/{short-description}
```

| Type | Example |
|---|---|
| `feat/` | `feat/DEV-1234/user-notifications` |
| `fix/` | `fix/DEV-5678/null-pointer-import` |
| `hotfix/` | `hotfix/DEV-999/critical-payment` |
| `chore/` | `chore/refactor-agent-setup` |

### Commits

```
{type}({scope}): {description}
```

Examples: `feat(DEV-1234): add notification preferences`, `fix(import): handle null values`

## Variable Naming

| Element | Convention | Example |
|---|---|---|
| Variables | camelCase | `$projectId`, `$importResult` |
| Constants | UPPER_SNAKE_CASE | `MAX_RETRIES`, `DEFAULT_TIMEOUT` |
| Array keys | snake_case | `['user_id' => 1, 'is_active' => true]` |
| Config keys | snake_case, dot notation | `config('import.max_rows')` |
| Env vars | UPPER_SNAKE_CASE | `APP_ENV`, `DB_HOST` |

## Module Naming

| Element | Convention | Example |
|---|---|---|
| Module directory | PascalCase | `app/Modules/Import/` |
| Module namespace | `App\Modules\{Name}\App\` | `App\Modules\Import\App\Services\` |
| Route prefix | kebab-case | `import`, `client-software` |
| Internal dirs | PascalCase | `App/`, `Routes/`, `Tests/` |

## Gotcha

- Consistency beats cleverness — match existing naming patterns even if you'd name it differently.
- The model tends to use abbreviations (`usr`, `msg`, `cfg`) — always spell out the full word.
- Database columns are snake_case, PHP properties are camelCase, class names are PascalCase — don't mix them.

## Do NOT

- Do NOT use abbreviations that aren't universally understood.
- Do NOT mix naming conventions within the same context.
- Do NOT use generic names like Manager, Helper, Utils without specificity.
